# Repository Guidelines

## Project Structure & Module Organization
- `src/index.ts` registers the `pyth` plugin and exports it.
- `src/config.ts` resolves defaults from env/config; `src/types.ts` defines shared types.
- `src/api/` wraps the Pyth Hermes API; `src/tools/` contains Tool definitions.
- `src/utils/` holds helpers for feed ID normalization and price formatting.
- `tests/` is for Vitest unit tests; `scripts/` hosts opt-in integration checks.
- `dist/` is generated by `tsup`; avoid manual edits.

## Build, Test, and Development Commands
- `npm run dev` watches and rebuilds with `tsup`.
- `npm run build` outputs `dist/` bundles and type definitions.
- `npm run typecheck` runs `tsc --noEmit` on `src/` and `tests/`.
- `npm run lint` and `npm run format` run Biome checks/formatting.
- `npm run test` runs Vitest; `npm run test:watch` keeps it in watch mode.
- `npm run test:integration` calls the Pyth API (requires network).
- `npm run example` runs the local demo script after a build.

## Coding Style & Naming Conventions
- TypeScript + ESM. Biome enforces 2-space indentation and 100-char lines.
- Tool methods use the `pyth_*` prefix and return `{ success, ... }` payloads.
- Use `camelCase` for functions/vars, `PascalCase` for types/classes, and lowercase file names.

## Testing Guidelines
- Keep unit tests deterministic and offline-friendly.
- Integration checks live in `scripts/integration-test.mjs`; gate network by env vars.
- Add tests for new price formatting or feed lookup behavior.

## Pyth-Specific Notes
- Default API base URL is `https://hermes.pyth.network`.
- Price feed IDs are hex; `normalizeFeedId` strips `0x` and lowercases for matching.
- If you add endpoints, update `src/api/endpoints.ts` and `README.md`.

## Commit & PR Guidelines
- Use Conventional Commits (`feat:`, `fix:`, `docs:`) with optional scopes.
- PRs should include a short summary, test results, and any new env vars or tools.
